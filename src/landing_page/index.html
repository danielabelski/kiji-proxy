Why Kiji

Every prompt is a data export.

Customer emails in support drafts. Real names in meeting notes. Credit card fragments in logs you ask an LLM to debug. Once sent, that data is outside your control. Kiji puts a checkpoint between your tools and the API — without changing how you work.

Automatic PII protection

ML-powered detection of 26 PII types — no regex lists to maintain, no manual redaction. The model and training data are open on HuggingFace, and you can fine-tune your own.

Zero code changes

Run it as a transparent proxy. On macOS, PAC auto-configures Safari and Chrome; for CLI tools you set HTTPS_PROXY once. Your existing apps, SDKs and agents keep working.

Everything stays local

Detection runs on your machine with ONNX Runtime — no external calls, no telemetry, no third-party PII service. The mapping between real and dummy values never leaves your device.

Built for compliance

Reduce GDPR, HIPAA and CCPA exposure when teams use AI for support, log analysis or development. A visual desktop UI shows exactly what was masked, when.

Quickstart

Protected in two minutes.

Download a release, start the proxy, point your traffic at it. That's the whole setup. 

# 1. Download the .dmg from GitHub Releases and drag to Applications
#    https://github.com/dataiku/kiji-proxy/releases

# 2. Start with sudo for automatic browser configuration (PAC)
sudo "/Applications/Kiji Privacy Proxy.app/Contents/MacOS/kiji-proxy"

# Safari & Chrome now route AI API traffic through Kiji automatically.
# 3. For CLI tools, point them at the proxy:
export HTTP_PROXY=http://127.0.0.1:8081
export HTTPS_PROXY=http://127.0.0.1:8081
# 1. Download and extract the standalone server
wget https://github.com/dataiku/kiji-proxy/releases/download/vX.Y.Z/kiji-privacy-proxy-X.Y.Z-linux-amd64.tar.gz
tar -xzf kiji-privacy-proxy-X.Y.Z-linux-amd64.tar.gz
cd kiji-privacy-proxy-X.Y.Z-linux-amd64

# 2. Run it
./run.sh

# 3. Route your tools through the proxy
export HTTP_PROXY=http://127.0.0.1:8081
export HTTPS_PROXY=http://127.0.0.1:8081
# Requires Go 1.25+, Node 20+, Python 3.13+, Rust
git clone https://github.com/dataiku/kiji-proxy.git
cd kiji-proxy

make electron-install
make setup-onnx

# Run the desktop app
make electron

Kiji listens as a forward proxy on :8080 and a transparent proxy on :8081. Full setup details in the Getting Started guide.

Examples

Put it in front of the tools you already use.

Anything that respects proxy environment variables — agents, SDKs, plain curl — can run through Kiji.

Coding agents

Claude Code, with a privacy layer

Coding agents read your real code, configs and logs — which often contain customer emails, names and keys. Route Claude Code through Kiji and PII is masked before any request leaves your machine, then restored in the responses you see. 

# 1. Start Kiji (macOS shown; see Quickstart for Linux)
sudo "/Applications/Kiji Privacy Proxy.app/Contents/MacOS/kiji-proxy"

# 2. Route Claude Code through the transparent proxy
export HTTP_PROXY=http://127.0.0.1:8081
export HTTPS_PROXY=http://127.0.0.1:8081

# 3. Work as usual — requests to the Anthropic API are masked in transit
claude "summarize the customer complaints in support_log.txt"

# Kiji's UI shows each masked token; originals are restored in the output.
API · curl

Any OpenAI-style request

Send a prompt containing an email through the proxy and watch it get masked before it leaves. 

export HTTPS_PROXY=http://127.0.0.1:8081

curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4","messages":[{"role":"user",
       "content":"My email is john@example.com"}]}'

# Sent to OpenAI: a dummy address.
# Returned to you: john@example.com, restored.
